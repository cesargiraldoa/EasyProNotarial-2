→ Cómo se traduce al wizard

Cada nodo es una regla ejecutable

El árbol no es un diagrama: es la estructura de datos que corre el wizard interactivo y el informe de cumplimiento. Cada nodo se modela así:

{
  id: "vendedor_estado_civil",
  pregunta: "Estado civil del vendedor",
  tipo: "estrategica",
  opciones: ["soltero","casado_sc","union_marital","divorciado","viudo"],
  reglas: [
    { si: "casado_sc || union_marital",
      efecto: "insertar_clausula:indagacion_vivienda_familiar",
      severidad: "obligatoria",  norma: "art. 6 Ley 258/1996" },
    { si: "afectado_vivienda_familiar == true",
      efecto: "exigir_firma:ambos_conyuges",
      severidad: "bloqueante",   norma: "art. 3 Ley 258/1996" }
  ]
}

Con esta forma, el motor de cláusulas ensambla el texto y el informe de cumplimiento evalúa bloqueantes, obligatorias y condicionales — todo trazable a su norma.
